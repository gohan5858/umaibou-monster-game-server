# Style and Conventions

This project appears to follow standard Rust conventions.

- **Formatting:** Use `cargo fmt` to format the code.
- **Linting:** Use `cargo clippy` to check for common mistakes and style issues.
- **Naming:** Follow the standard Rust naming conventions (e.g., `snake_case` for variables and functions, `PascalCase` for types).
- **Docstrings:** Add docstrings to public functions and types to explain their purpose.
