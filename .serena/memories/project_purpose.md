# プロジェクトの目的

## 概要

3Dリアルタイム対戦ゲームサーバー（2人対戦型）

## 主な機能

- **マッチングシステム**: REST APIによるプレイヤーマッチング（ID生成、参加、成立通知）
- **リアルタイム通信**: WebSocketによる60Hzのゲーム状態配信
- **ゲーム進行管理**: リアルタイム操作入力、状態計算、勝敗判定
- **3Dモデル管理**: 3Dモデルのアップロード機能

## 非機能要件

- 60Hzゲーム状態更新（tokio::intervalで16.67ms間隔）
- 1000組同時処理対応
- REST API応答時間 < 100ms

## アーキテクチャ特徴

- Actixアクターモデルによるメッセージ駆動の状態管理
- Arc<Mutex<HashMap>>による並行処理
- mpsc::unbounded_channelによる非同期メッセージ配信
